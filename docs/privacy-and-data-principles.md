# Privacy and Data Principles

CareBridge Canada must be designed as a privacy-first healthcare product.

The MVP uses synthetic data only.

No real patient data should be entered, stored, committed, uploaded, exported, or used in demos during early development.

## MVP data rules

The MVP must not collect:

- real patient names
- health card numbers
- real lab results
- real prescriptions
- real discharge documents
- real appointment transcripts
- real personal health information

Use synthetic demo personas only.

## Privacy principles

CareBridge should be designed around:

- patient ownership
- data minimization
- clear consent
- transparent AI outputs
- source-grounded summaries
- exportable records
- caregiver permission controls
- no hidden provider access
- no provider account requirement in MVP
- no government portal integration in MVP

## Evidence Cards

Generated outputs should show what they were based on.

Examples:

- appointment note
- pasted instruction
- pending referral task
- medication list
- province guidance
- family translation request

The app should avoid mystery AI answers.

## Future security requirements

Before real patient use, CareBridge would need:

- secure authentication
- encrypted data storage
- audit logs
- role-based caregiver access
- secure deletion
- privacy policy
- consent records
- legal/privacy review
- healthcare compliance review
- secure backend deployment
